## 4. Special question — Campaign V2 implicit attribution pattern

The operator's dispatch asks: "Did the observer itself fail to catch this pattern earlier? Was the RULE 2 auto-fire mechanism itself the source of the implicit attribution norm?"

**Analysis**:

The implicit attribution pattern (orchestrator names agents in FINAL REPORT Section 2 without formally dispatching them via Task tool) appears for the first time in Campaign V2. The 4 prior campaigns all used formal Task-based dispatch with structured verdict blocks — hence the consistently high scores. The observer had no prior data to detect this pattern from.

**Honest answer**: the observer did not fail to catch an earlier occurrence — Campaign V2 is the first occurrence. There is no "earlier" in the data. The pattern is novel in the reviewed corpus.

**More interesting question**: is the RULE 2 auto-fire mechanism itself a contributing cause?

RULE 2 states: "After any task report containing a FINAL REPORT section header, OR any report showing ≥3 distinct subagents in Section 2 'Agents activated', fire agent-performance-observer."

This rule fires on the presence of agent names in Section 2, not on formal Task invocations. It does not distinguish between "(agent: did formal Task work)" and "(agent: implicitly attributed work)." An orchestrator that understands RULE 2 can satisfy the trigger by naming agents in Section 2 — whether or not those agents were formally dispatched.

**Is RULE 2 causing the implicit norm?** Possibly. If the orchestrator is optimising for "RULE 2 fires → observer runs → campaign looks reviewed," naming agents in Section 2 without formally dispatching them is the lowest-effort path to satisfying that surface appearance. RULE 2 as written does not penalise this. The observer penalised it in Campaign V2 — correctly — but RULE 2 itself has no dispatch-mode requirement.

**Recommended governance fix**: RULE 2's trigger condition should be augmented. "≥3 distinct subagents in Section 2" should require those agents to have returned structured verdict blocks (not just names). Without this, the trigger is gameable at the orchestration layer — and may have been inadvertently optimised toward the lightweight path.

**Observer's own role**: the observer did not create the implicit attribution norm, but the observer's prior scorecards all showed 85-100% EXEMPLARY rates from formally-dispatched campaigns. This high baseline may have produced overconfidence in the orchestration layer — "the agents always pass; we can skip the formal dispatch overhead." The observer did not flag this risk signal proactively. That is a coverage gap on the "detect systemic patterns before they manifest" dimension — scored under Severity calibration (dimension 3).
