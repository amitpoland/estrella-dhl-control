## 0. Incident Timeline and Failure Modes

Sequence of events on AWB 2315714531 / wFirma PZ 4/6/2026 (document 189364835). Dates are 2026-06-18
unless noted; the booked PZ (`wfirma_pz_created`) timeline event is timestamped `2026-06-18T06:09:11Z`.

1. **Booking.** A wFirma PZ was created for the shipment (document 189364835), booking net **2,280.14 PLN**
   from the then-current line prices (unit 30.45 / 67.61). The pointer was written to
   `audit.wfirma_export` by `global_pz_push`.
2. **Class A surfaces (landed-cost incompleteness).** This is an image-only invoice. The image-only
   ingestion path did not supply Freight + Insurance to the engine, so `process_batch()` computed an
   incomplete landed cost and customs value resolved to a fabricated **CIF = 0.00** instead of an
   honest `UNKNOWN`. Corrected landed-cost authority (`pz_rows.json`) is net **2,736.87 PLN** (unit
   36.55 / 81.16) — the booked document is low by **456.73 PLN (+20.0%)**. Failure mode: silent,
   no error raised; downstream action gates read the fabricated zero and produced false states.
3. **Class B surfaces (external-reference loss).** Each Run PZ rebuilt `audit.json` from engine-only
   output. `merge_regenerated_audit()` overlays only keys in `PRESERVED_KEYS`; `wfirma_export` was in
   neither `PRESERVED_KEYS` nor the engine output, so **every regeneration nulled
   `audit.wfirma_export`** — four post-correction regenerations left it null. The link to document
   189364835 survived only in the append-only `timeline`.
4. **Detection.** The operator observed the wFirma value gap and a null `wfirma_export` after Run PZ.
   Detection was manual — no automated signal fired (the absence of that signal is the open Rule 3 gap, §3/§5).
5. **Resolution — Class A.** PR #648 (`8024c50`) allocates Freight + Insurance into PZ net on the
   image-only path and routes customs value through the tri-state CIF authority. Found already
   deployed on-disk during this incident's gate.
6. **Resolution — Class B.** PR #652 (`03ffce9`) adds `wfirma_export` to `audit_merge.PRESERVED_KEYS`.
   Deployed via scoped single-file sync after the 7-agent gate (§2); verified
   `wfirma_export in PRESERVED_KEYS == True` in the running service.
7. **Pointer recovery.** Post-deploy the pointer read null (wiped by regenerations that ran under the
   pre-restart code). `audit_persist.reconcile_from_timeline()` restored
   `wfirma_export.wfirma_pz_doc_id = 189364835` from the `wfirma_pz_created` event. Durable post-#652.
8. **Open.** The booked wFirma **value** (2,280.14 → 2,736.87 PLN) is not yet corrected — operator-owned
   accounting decision (§1, §5). No wFirma write has been performed.

**Impact scope.** One shipment (AWB 2315714531). Class A under-stated landed cost and mis-stated the
customs basis (fabricated CIF 0.00) for image-only invoices as a class. Class B silently dropped the
audit→wFirma document link on every regeneration. No other shipment is known affected; the fixes are
class-level (every ingestion path / every external reference), not shipment-specific.

## 1. Governance Decisions

Decisions made as a direct result of this incident. These are policy, not guidance.

**Prohibited.**
- No API price-edit of an already-booked wFirma PZ document. The wFirma client exposes no validated
  edit, update, or delete for `warehouse_document_p_z` (only `create_warehouse_pz`,
  `fetch_warehouse_pz` read, `find_warehouse_pz_by_number`). `global_pz_push.py` is create-only;
  `CANCEL_AND_RECREATE` is out of scope. No new API price-edit path may be built unless a wFirma
  **sandbox** first proves safe editing of a **posted** PZ.
- No silent auto-correction of a value mismatch between the PZ engine and a booked wFirma document.
  Mismatches are surfaced as a comparison workflow, never reconciled invisibly.

**Approved path.**
- Correcting a booked PZ is an operator action in wFirma: either (a) manual line-price correction in
  the wFirma UI, or (b) cancel/delete the document in wFirma and recreate through the gated
  `global_pz_push` create path from the corrected `pz_rows.json`. Any correction records the
  old → new document linkage in the `timeline` / audit.
- All wFirma writes require explicit operator approval. The deploy-guard hook blocks agent writes
  into `C:\PZ`; production sync and service restart are operator-executed.

**Rationale.** A booked wFirma PZ is the accounting system of record and is locked once booked. The
EJ integration has no proven-safe write to mutate it, and the integration precedent rejects price
edits on posted documents. Until a sandbox proves otherwise, manual correction or cancel/recreate is
the only safe path. The local audit and engine are already correct; the only divergence is in the
external system, which the business — not the software — must reconcile.

## 3. Authority Rules Established

### Established and working

**Rule 1 — Landed cost is complete and computed by one engine across every ingestion path.**
- **Covers:** PZ net composition — FOB, Freight, Insurance, Duty — and the customs value (CIF).
- **Scope:** Every invoice ingestion path, text-parsed and image-only. `process_batch()` is the only
  calculation path.
- **Prevents:** Understated landed cost and fabricated `CIF = 0.00` on degraded ingestion paths,
  and the false customs blocks / false readiness states those produce.
- **In action:** An image-only invoice now allocates Freight + Insurance into PZ net by value, and
  resolves CIF to `RESOLVED` / `DECLARED_ZERO` / `UNKNOWN` — never a silent zero. Before #648, the
  same invoice produced PZ net missing F+I and a customs value of `0.00`.

**Rule 2 — External-reference authority is explicitly preserved across audit regeneration.**
- **Covers:** Audit blocks holding a reference to an external system's document — `wfirma_export`
  (booked-PZ id), DHL labels, customs/SAD/MRN/ZC429 references, WorkDrive resource IDs.
- **Scope:** Every Run PZ, which rebuilds `audit.json` from engine-only output and overlays only the
  keys in `audit_merge.PRESERVED_KEYS`. The merge is asymmetric — a meaningful regenerated value
  always wins, so preservation never blocks a legitimate update.
- **Prevents:** Silent loss of the link between a shipment and its booked external document.
- **In action:** After #652, a Run PZ that omits `wfirma_export` keeps the existing pointer to
  document 189364835. Before #652, every regeneration nulled it; the link survived only in the
  append-only `timeline`.

### Missing — future work

**Rule 3 (not yet established) — Reconciliation authority between the engine and wFirma.**
- **Would cover:** Divergence between the recalculated PZ result and the booked wFirma document.
- **Scope:** All shipments with a booked PZ, evaluated continuously and before a period closes.
- **Prevents:** The root-cause category this incident exposed — PZ recalculates, wFirma holds a
  booked document, values diverge, no one notices. Today this depends on a human investigating.
- **Status:** Not built. This is the primary strategic project (§5). Rules 1 and 2 guarantee correct
  value and a preserved link; neither makes the divergence observable. That gap is Rule 3.

2. **Booked-PZ correction workflow.** Fills the operational gap between detection and correction.
   Operationalizes manual/recreate; requires `CANCEL_AND_RECREATE` behind
   `WFIRMA_CORRECTION_PUSH_ALLOWED` (default OFF), the five background-automation safety properties,
   and automatic old → new linkage. Depends on project 1 to identify targets.
3. **wFirma API-edit research.** Fills no current gap and is conditional. Hard-gated on a wFirma
   sandbox proving safe edit of a posted PZ. Closed until that proof exists; correction stays manual
   or recreate.
